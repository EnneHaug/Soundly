Class `AlarmEngine`:
- Private fields:
  - `ac: AudioContext | null = null`
  - `phase: AlarmPhase = 'idle'`
  - `timers: TimerHandle[] = []`
  - `keepaliveOsc: OscillatorNode | null = null`
  - `phase3RampGain: GainNode | null = null`
  - `phase3SwellOsc: OscillatorNode | null = null`
  - `phase3LoopTimer: ReturnType<typeof setInterval> | null = null`
  - `phaseCallback: PhaseChangeCallback | null = null`

- `getPhase(): AlarmPhase` — returns current phase (read-only accessor)

- `onPhaseChange(cb: PhaseChangeCallback): void` — registers callback. Only one callback at a time (last one wins). React will use this with a ref.

- Private `setPhase(newPhase: AlarmPhase): void`:
  - Sets `this.phase = newPhase`
  - Calls `this.phaseCallback?.(newPhase)`

- `async start(config: AlarmConfig): Promise<void>`:
  - Call `validateConfig(config)` (throws on invalid)
  - If `this.phase !== 'idle'`, throw Error('AlarmEngine already running')
  - `this.ac = await getAudioContext()` (MUST be called from user gesture context)
  - Start keepalive: `this.keepaliveOsc = startKeepalive(this.ac)` (per AUD-04)
  - Calculate absolute fire times from `Date.now()`:
    - `phase1FireAt = Date.now() + config.phase1DurationMs`
    - `phase2FireAt = phase1FireAt + config.phase2DurationMs`
    - `phase3FireAt = phase2FireAt + config.phase2to3GapMs` (configurable gap, default 10s per DEFAULT_CONFIG)
  - Schedule Phase 1 transition:
    ```
    this.timers.push(scheduleAt(phase1FireAt, () => {
      this.setPhase('phase1');
      strikeBowl(this.ac!, 1.0); // full volume singing bowl
    }));
    ```
  - Schedule Phase 2 transition:
    ```
    this.timers.push(scheduleAt(phase2FireAt, () => {
      this.setPhase('phase2');
      // Phase 2 triggers vibration — handled in Phase 2 (Background Reliability)
      // The singing bowl's natural decay handles D-04 overlap-then-fade
    }));
    ```
  - Schedule Phase 3 transition:
    ```
    this.timers.push(scheduleAt(phase3FireAt, () => {
      this.enterPhase3(config.phase3RampDurationMs);
    }));
    ```
  - Set phase to 'idle' remains (we're in the waiting/countdown period before phase1 fires)
  - Actually transition to a "running" conceptual state — but the AlarmPhase type starts at 'idle' until phase1 fires. Keep phase as 'idle' until the first phase triggers. (The UI in Phase 3 will show countdown based on timer, not phase.)

  Wait — re-examining the state machine: `idle` should mean "not running". When the user presses Start, we need a state that means "timer running, waiting for Phase 1". Add to AlarmState.ts: the `idle` state covers "not started". Once `start()` is called, we stay in `idle` until phase1 fires. This is acceptable — the phase represents the alarm escalation phase, not the timer state. The UI will track "is timer running" separately.

- Private `enterPhase3(rampDurationMs: number): void`:
  - `this.setPhase('phase3')`
  - Create ramp: `this.phase3RampGain = createPhase3Ramp(this.ac!, rampDurationMs / 1000)` (convert ms to seconds)
  - Start initial swell: `this.phase3SwellOsc = startPhase3Swell(this.ac!, this.phase3RampGain)`
  - Set up looping: every 3.2 seconds, stop old swell and create new one (factory pattern per D-05):
    ```
    this.phase3LoopTimer = setInterval(() => {
      try { this.phase3SwellOsc?.stop(); } catch {}
      this.phase3SwellOsc = startPhase3Swell(this.ac!, this.phase3RampGain!);
    }, 3200);
    ```

- `stop(): void` — cancels everything, returns to idle:
  - Cancel all timers: `this.timers.forEach(t => t.cancel()); this.timers = [];`
  - Stop keepalive: `if (this.keepaliveOsc) { stopKeepalive(this.keepaliveOsc); this.keepaliveOsc = null; }`
  - Stop Phase 3 loop: `if (this.phase3LoopTimer) { clearInterval(this.phase3LoopTimer); this.phase3LoopTimer = null; }`
  - Stop Phase 3 swell: `try { this.phase3SwellOsc?.stop(); } catch {} this.phase3SwellOsc = null;`
  - Fade out Phase 3 ramp: `if (this.phase3RampGain && this.ac) { fadeOutGain(this.phase3RampGain, this.ac); this.phase3RampGain = null; }`
  - `this.setPhase('idle')`

- `dismiss(): void` — same cleanup as stop but transitions to 'dismissed':
  - Same cleanup logic as stop()
  - `this.setPhase('dismissed')`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | AlarmConfig durations | mitigate | `validateConfig()` rejects durations <= 0 or > 14,400,000ms (4 hours). Prevents nonsensical timer values and potential integer overflow. |
| T-01-05 | Denial of Service | Phase 3 swell loop | mitigate | setInterval at 3200ms creates new OscillatorNode each cycle. `stop()` and `dismiss()` both clear the interval and stop the oscillator. Without cleanup, nodes would accumulate. Both exit paths (stop/dismiss) perform full cleanup. |
| T-01-06 | Denial of Service | Multiple start() calls | mitigate | `start()` throws if `this.phase !== 'idle'`. Prevents creating duplicate timers and keepalive oscillators. |
</threat_model>
