```typescript
// SPA navigation fallback — serves cached index.html for all navigation requests.
// Belt-and-suspenders: precacheAndRoute handles exact URL matches, but
// NavigationRoute catches browser navigation to paths not in the precache manifest
// (e.g., direct URL entry, refresh on a deep link if one is ever added).
const navHandler = createHandlerBoundToURL('/index.html');
registerRoute(new NavigationRoute(navHandler));
```

This requires installing `workbox-routing` as a dev dependency:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SW scope | Service worker controls all navigation within origin; must not intercept cross-origin requests |
| Notification click | SW notificationclick handler opens windows — must only open same-origin URLs |
| Precache manifest | Build-time injection of asset list — must not include sensitive files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | SW notificationclick | accept | Handler only focuses existing same-origin windows or opens "/". No user input processed. Low risk — attacker cannot trigger notifications without SW registration. |
| T-04-02 | Tampering | Precache manifest | accept | Build-time generated by Workbox from Vite output. No user input. Served over HTTPS in production. |
| T-04-03 | Information Disclosure | SW cache | accept | Caches only public static assets (JS, CSS, HTML, icons). No PII, no API keys, no user data. App has no persistent storage. |
| T-04-04 | Denial of Service | NavigationRoute | mitigate | NavigationRoute only matches navigation requests (document type). Does not intercept API calls, images, or other subresources. Workbox default behavior is safe. |
| T-04-05 | Elevation of Privilege | SW scope | accept | SW scope is `/` (same as start_url). No cross-origin requests. injectManifest does not auto-call skipWaiting/claim (per D-01), so new SW versions do not hijack active sessions. |
</threat_model>
